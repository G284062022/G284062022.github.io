学習スタイル・方針 / How I Study

1. 広く全体像をつかむ

まずはシラバスと公開問題をベースに、「どんな分野が、どのくらいの比率で出るのか」を把握しました。 その上で、苦手な分野（例：マネジメント系、セキュリティ用語など）をリストアップし、 毎日少しずつ潰していくスタイルで学習しています。

  • 午前対策：用語・計算問題を中心に、過去問のパターンを整理
  • 午後（科目B）対策：アルゴリズムとデータ構造、ネットワーク、DB を重点的に演習

2. 実務・自分の開発と結びつける

単に暗記するのではなく、OBD 車両診断システムやスマートスーツケースなど、 自分の開発プロジェクトと関連付けて理解するようにしています。

  • ネットワーク：REST API や HTTP の知識を、自作 Web アプリの設計に反映
  • データベース：正規化・トランザクションの考え方を、自分の DB 設計で実験
  • セキュリティ：認証・セッション管理をポートフォリオの Web アプリに適用

練習問題① アルゴリズム・データ構造

基本情報の午後問題でも頻出の分野です。ここでは、自分でまとめた練習問題と簡単な解説の一部を載せています。

Q1（探索アルゴリズム）
昇順に整列済みの配列に対し、二分探索を用いるときの計算量のオーダとして適切なものはどれか。
① O(1)　② O(n)　③ O(log n)　④ O(n log n)
答え：③ O(log n)
毎回探索範囲を 1/2 にしていくため、要素数 n に対しておおよそ log2n 回の比較で済みます。
Q2（スタックとキュー）
先入れ先出し（FIFO）のデータ構造として適切なものはどれか。
① スタック　② キュー　③ ヒープ　④ ハッシュテーブル
答え：② キュー
スタックは LIFO（後入れ先出し）、キューは FIFO（先入れ先出し）という動作原理の違いを意識して覚えています。
Q3（計算量の比較）
n が十分に大きいとき、次のうち最も効率が良い（計算量が小さい）ものはどれか。
① O(n^2)　② O(n log n)　③ O(n)　④ O(log n)
答え：④ O(log n)
オーダの大小関係を感覚的に把握しておくことで、ソートアルゴリズム選択や設計時の判断がしやすくなります。

練習問題② ネットワーク・セキュリティ

Web アプリ開発と直結する分野なので、自分のポートフォリオ案件（OBD システム等）をイメージしながら学んでいます。

Q4（OSI 参照モデル）
次のうち、OSI 参照モデルの第 4 層（トランスポート層）に属するプロトコルはどれか。
① IP　② TCP　③ HTTP　④ Ethernet
答え：② TCP
IP は第 3 層、HTTP は第 7 層、Ethernet は第 2 層。 レイヤと代表プロトコルの対応を、ネットワーク設計やトラブルシュートに活かせるよう整理しています。
Q5（暗号方式）
共通鍵暗号方式と比較したときの公開鍵暗号方式の特徴として、最も適切なものはどれか。
① 暗号化・復号が高速である。
② 鍵配送の問題を軽減できる。
③ 鍵が漏えいしても安全である。
④ 第三者が復号鍵を生成できる。
答え：② 鍵配送の問題を軽減できる
公開鍵暗号は処理は重いですが、「公開鍵は誰に見られても良い」という性質を利用し、 秘密鍵を共有せずに安全な通信を開始できる点がポイントです。
Q6（Web セキュリティ）
Web アプリケーションにおいて、SQL インジェクション対策として適切なものはどれか。
① 入力値を HTML エスケープする。
② 入力値を暗号化して保存する。
③ プレースホルダ付きのプリペアドステートメントを用いる。
④ Cookie に Secure 属性を付ける。
答え：③ プレースホルダ付きのプリペアドステートメントを用いる
文字列連結による SQL の生成を避け、バインド変数を用いる設計を自分の Web アプリでも徹底するようにしています。

練習問題③ データベース・システム設計

Q7（正規化）
1 つの表に「顧客ID」「顧客名」「注文ID」「注文日」がすべて入っており、 同じ顧客が複数の注文を行うと行が重複する。この表に対して適切な正規化方針はどれか。
① 列を増やして 1 行にすべての注文を格納する。
② 顧客と注文を別の表に分割し、顧客IDで関連付ける。
③ 顧客名を削除する。
④ 注文日を別表に移す。
答え：② 顧客と注文を別の表に分割し、顧客IDで関連付ける
「顧客：注文 = 1：多」の関係を意識し、エンティティごとに表を分けることが第 3 正規形への基本的なステップです。
Q8（トランザクション）
トランザクションの ACID 特性のうち、「途中の状態が外部から見えない」ことを表すものはどれか。
① 原子性（Atomicity）
② 一貫性（Consistency）
③ 独立性（Isolation）
④ 永続性（Durability）
答え：③ 独立性（Isolation）
自作の Web アプリでも、複数ユーザの同時更新やロールバックを意識した実装を行うように意識しています。

応用・実務へのつながり / Applications in My Projects

プロジェクトへの活用例

  • OBD 車両診断システム： ネットワークと API の知識をもとに、Vue フロントエンドと Spring Boot バックエンド間の REST 通信設計・エラーハンドリング・ログ設計を整理。
  • スマートスーツケース： 組込みシステムの知識を活かし、Raspberry Pi ＋ Arduino の役割分担や、 センサデータを扱う際のリアルタイム性・信頼性について検討。
  • 研究活動： データベース・統計・セキュリティの知識を、絵文字共起分析や Web アプリの研究プロトタイプに適用。

試験勉強で得た知識を「暗記」で終わらせず、自分の開発や研究テーマの設計に落とし込むことで、 技術の理解を深めるようにしています。
