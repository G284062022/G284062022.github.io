Frontend & Backend Basics for Web Development

概要 / Overview

Web アプリケーションは大きく「フロントエンド」と「バックエンド」に分かれており、 それぞれが役割を分担しながら 1 つのサービスとして動作しています。 このページでは、開発初心者の方に向けて、前後端の役割や技術スタック、データの流れを シンプルに解説します。

フロントエンド バックエンド API HTTP DB / SQL REST JSON

フロントエンドとは？ / What is Frontend?

ユーザーと直接触れる「見える部分」

フロントエンドは、ユーザーがブラウザ上で直接触れる「画面側」の部分です。 画面のレイアウト、ボタンやフォームの動き、グラフ表示、アニメーションなどを担当します。

  • HTML：ページの構造（見出し・段落・フォームなど）を定義
  • CSS：デザインやレイアウト（色・余白・レスポンシブ対応）を調整
  • JavaScript：ボタンのクリック処理や動的な画面更新を実装

よく使われるフレームワーク

大規模な画面や複雑な UI を効率よく作るために、フレームワーク・ライブラリが使われます。

React Vue.js Angular TypeScript
  • MDN Web Docs（HTML / CSS / JS） https://developer.mozilla.org/
  • React 公式サイト https://react.dev/
  • Vue.js 公式サイト https://vuejs.org/

バックエンドとは？ / What is Backend?

画面の裏側で動く「見えない部分」

バックエンドは、ブラウザの裏側で動作するサーバー側の処理を担当します。 データベースへのアクセス、業務ロジックの実行、認証・認可、外部サービスとの連携などを行います。

  • ブラウザからのリクエスト（例：ログイン、登録、検索）を受け取る
  • 必要な処理やデータベース操作を行う
  • 結果を JSON などの形式でフロントエンドへ返す

代表的なバックエンド技術

言語やフレームワークによって書き方は異なりますが、基本的な役割は共通です。

Java / Spring Boot Python / Django Node.js / Express Ruby on Rails
  • Spring Boot 公式ドキュメント https://spring.io/projects/spring-boot
  • Django 公式サイト https://www.djangoproject.com/
  • Express（Node.js）公式 https://expressjs.com/

データベースと API / Database & API

データベース（DB）の役割

ユーザー情報、商品情報、ログデータなどを永続的に保存するのがデータベースです。 バックエンドは SQL などを使って DB とやり取りします。

MySQL PostgreSQL SQLite MongoDB
  • PostgreSQL https://www.postgresql.org/docs/
  • MySQL https://dev.mysql.com/doc/

API と HTTP リクエスト

フロントエンドとバックエンドは、通常「HTTP」というプロトコルを使って通信します。 REST API では、URL と HTTP メソッド（GET / POST / PUT / DELETE など）を組み合わせて データの取得・作成・更新・削除を行います。

  • フロントエンド：fetch や axios で API を呼び出す
  • バックエンド：リクエストを受け取り、必要な処理を実行
  • レスポンス：JSON 形式で結果を返し、画面に反映
  • MDN – HTTP の基礎 https://developer.mozilla.org/docs/Web/HTTP

前後端のつながり / How Frontend & Backend Work Together

1 つのリクエストの裏側で起きていること

  • ユーザーがブラウザでフォームに入力し、「送信」ボタンを押す。
  • フロントエンドが入力値をチェックし、必要であればエラー表示。
  • 問題がなければ、API（例：/api/users）に HTTP リクエストを送信。
  • バックエンドがリクエストを受け取り、バリデーション・認証・DB 操作を実行。
  • 処理結果（成功／エラー）を JSON として返す。
  • フロントエンドが結果を受け取り、「登録完了」のメッセージや次の画面を表示。

実際のプロジェクトでは、ログ記録、エラーハンドリング、セキュリティ対策（認証・CSRF・CORS など）も 含めて設計することが重要です。

学習のステップ / How to Start Learning

  1. まずは HTML / CSS / JavaScript の基礎を MDN などで一通り学ぶ。
  2. 小さな静的サイトを作り、フォームや簡単な動き（タブ切り替え等）を実装してみる。
  3. バックエンドフレームワーク（例：Spring Boot や Django）で、シンプルな API を作る。
  4. フロントエンドから API を呼び出して、JSON データを画面に表示してみる。
  5. ログイン機能、データベース連携、エラーハンドリングなどを追加し、少しずつ Web アプリらしくしていく。

小さな「動くもの」を作りながら、疑問が出てきたら公式ドキュメントや書籍で調べていく、 というサイクルが成長の近道です。
