プロジェクト概要 / Overview

本プロジェクトは、OBD（On Board Diagnostics） の挙動を Web 上で再現する 車両診断シミュレーター です。 フロントエンドから疑似 ECU にアクセスし、エンジン回転数・車速・冷却水温・ バッテリー電圧・DTC（故障コード）などの情報をリアルタイムに可視化します。

面接では、「API 設計」「フロントエンドとバックエンドの連携」、 「状態管理」 を説明するためのデモとして利用できます。

使用技術 / Tech Stack

フロントエンド

  • Vue 3 + Vite
  • TypeScript（または JavaScript）
  • Axios による REST API 通信
  • レスポンシブデザイン（スマートフォン対応）

バックエンド

  • Java 17 / Spring Boot
  • OBD データの疑似生成ロジック
  • RESTful API 設計（/api/obd/live, /api/obd/dtc など）
  • Docker / AWS（デプロイしていれば）

主要機能 / Key Features

1. ライブデータモニタリング

一定間隔でバックエンド API から OBD データを取得し、 エンジン回転数・車速・冷却水温・スロットル開度 などを ダッシュボード上にリアルタイム表示します。

2. DTC（故障コード）シミュレーション

ランダムまたは条件付きで故障コードを発生させ、 フロントエンドで 警告表示・コード一覧・簡易説明 を表示します。 「DTC 消去」ボタンでコードをリセットすることも可能です。

3. 診断フローの可視化

OBD ポート → ECU → Web API → ブラウザ UI までの流れを 図とテキストで説明し、面接でアーキテクチャを説明しやすくしています。

4. モード切り替え

「通常走行」「異常発生」「DTC クリア」などのシナリオを切り替え、 それぞれの状態で値の変化や UI 反応を確認できます。（実装予定項目として書いてもOK）

システム構成 / Architecture

実際には画像や図を挿入するとわかりやすいです。 下のダミー画像をあなたのアーキテクチャ図に差し替えてください。

  • フロントエンド：Vue コンポーネントが一定間隔で API をポーリング
  • バックエンド：Spring Boot が疑似 ECU としてセンサ値を生成し JSON で返却
  • 状態管理：診断中フラグ、DTC リスト、グラフ用データなどをフロント側で管理

担当範囲 / My Role

  • 要件整理と画面設計（ワイヤーフレーム作成）
  • フロントエンドのコンポーネント設計・実装
  • Spring Boot による API 実装とエラーハンドリング設計
  • GitHub でのバージョン管理、デプロイ設定（GitHub Pages / AWS 等）

工夫した点・学び / What I Learned

面接で話せるように、箇条書きで 3〜4 点まとめておくと便利です。

  • API のレスポンス設計を意識し、フロントエンドから扱いやすい JSON にしたこと。
  • 実車ではなくシミュレーターであっても、OBD 診断の流れを意識して値の変化を設計したこと。
  • 状態管理・ローディング・エラー表示など、ユーザビリティを意識した UI 実装。
  • GitHub Pages / AWS へのデプロイを通じて、開発〜公開までの一連の流れを経験したこと。
